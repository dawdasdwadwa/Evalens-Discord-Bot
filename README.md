# Evalens Discord Bot

Discord-бот сервера Evalens. Плоская структура — все `.py`-файлы лежат в
корне репозитория (без `cogs/`, `config/`, `utils/`), деплой на Railway.

## Функциональность

1. **Приветствие новых участников** (`welcome.py` + `image_generator.py`)
   При заходе участника автоматически выдаётся роль и в канал приветствия
   отправляется сгенерированная через Pillow PNG-карточка с пингом
   пользователя.

2. **Верификация по кнопке** (`verification.py`)
   Панель верификации публикуется автоматически при каждом старте/редеплое
   бота (старые сообщения бота в канале чистятся перед публикацией новой
   панели). Успешные верификации логируются отдельно.

3. **Логи приглашений** (`invite_logs.py`)
   При заходе участника определяется, по какому инвайту он зашёл (сравнение
   счётчиков использований до/после), и логируется: кто зашёл, чей инвайт,
   код, число использований, когда и кем создана ссылка. Для vanity-ссылок
   и виджета Discord API не отдаёт эту информацию — бот честно пишет
   "не удалось определить". Требует право бота **Manage Server**.

4. **Модерация** (`moderation.py`)
   Слэш-команды `/ban`, `/unban`, `/mute`, `/unmute`, `/server`. Доступны
   только участникам с ролями из `STAFF_ROLE_IDS`.
   - `/ban` — бан с причиной и сроком (выбор из списка: 1 час / 1 день /
     3 дня / 7 дней / 30 дней / навсегда). Временные баны хранятся в
     `temp_bans.json` и снимаются автоматически фоновой задачей.
   - `/mute` — мьют через нативный Discord timeout (макс. 28 дней — это
     лимит самого Discord API, не бота).
   - `/unban`, `/unmute` — снятие бана/мьюта с указанием причины.
   - `/server` — статистика сервера (участники, каналы, роли, бусты).
   - **Автомодерация**: сообщение со ссылкой-приглашением на чужой Discord
     от не-стаффа удаляется, автору выдаётся мьют на 10 минут с уведомлением
     в ЛС, действие логируется.
   - Баны/разбаны логируются в `BAN_LOG_CHANNEL_ID`, мьюты/размьюты — в
     `MUTE_LOG_CHANNEL_ID`.

## Структура проекта

```
.
├── bot.py               # точка входа, грузит коги
├── settings.py          # все настройки через переменные окружения
├── welcome.py           # приветствие + автовыдача роли при входе
├── verification.py      # автопубликация панели, кнопка, логи верификации
├── invite_logs.py       # определение инвайта и логирование
├── moderation.py        # /ban /unban /mute /unmute /server + автомодерация
├── image_generator.py   # генерация PNG-карточки (Pillow)
├── Poppins-*.ttf         # шрифты для карточки
├── requirements.txt
├── Procfile
├── .env.example
└── .gitignore
```

## Установка

```bash
pip install -r requirements.txt
cp .env.example .env
```

Заполни `.env` (или, для продакшена, задай эти же переменные в
Railway → Variables — **не в самом репозитории**):

- `DISCORD_TOKEN` — токен бота (Discord Developer Portal → Bot → Token)
- `VERIFIED_ROLE_ID` — роль, выдаваемая после верификации (обязательно)
- `UNVERIFIED_ROLE_ID` — роль "новичок", снимается после верификации (опционально)

Остальные переменные (ID каналов и ролей) уже захардкожены как значения
по умолчанию в `settings.py` и переопределять их не обязательно —
но при желании их тоже можно вынести в `.env` (см. `.env.example`).

## Права и intents (Discord Developer Portal → Bot)

Privileged Gateway Intents — включить оба:
- **SERVER MEMBERS INTENT** — обязательно для приветствия и автовыдачи роли
- **MESSAGE CONTENT INTENT** — обязательно для автомодерации ссылок-приглашений в `moderation.py`

Права роли бота на сервере (роль бота должна быть **выше** ролей, которыми
она управляет — верификационной, замьюченных/забаненных участников):
- `Manage Roles`
- `Manage Server` (для `invite_logs.py`)
- `Ban Members`, `Moderate Members` (для `moderation.py`)
- `Send Messages`, `Embed Links`, `Attach Files`, `Manage Messages`
- `Use Application Commands`

## Запуск

```bash
python bot.py
```

## Деплой на Railway

1. Подключить репозиторий к Railway
2. Добавить переменные окружения в настройках проекта
3. Railway автоматически определит `requirements.txt` и запустит `Procfile` (`worker: python bot.py`)

Если используются временные баны (`/ban` со сроком), для надёжности
рекомендуется подключить Railway Volume — иначе `temp_bans.json`
сбрасывается при каждом редеплое и текущие временные баны не снимутся
автоматически (придётся `/unban` вручную).
